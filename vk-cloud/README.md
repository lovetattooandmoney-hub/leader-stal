# Заявки с сайта → почта через VK Cloud

Здесь код **облачной функции** (Node.js), которую вы разворачиваете в **[VK Cloud](https://cloud.vk.com/)**.  
Сайт на GitHub Pages шлёт форму на эту функцию; она отправляет **письмо на почту через SMTP** (Яндекс, Mail.ru и т.д.).  

**Не используется:** SMS, сообщения ВКонтакте, сторонние form-сервисы за рубежом.

Интерфейс и шаги в консоли VK Cloud со временем могут меняться — ориентируйтесь на актуальную документацию: [cloud.vk.com/docs](https://cloud.vk.com/docs).

## Что сделать в общих чертах

1. Войти в **VK Cloud**, создать каталог/проект (если ещё нет).
2. Найти сервис **Serverless / функции / Function** (название в консоли может отличаться) и создать **HTTP-функцию** на **Node.js 18+**.
3. Точка входа: **`index.handler`** (файл `index.js`, экспорт `handler`).
4. Загрузить архив **содержимого** папки `function/`: `index.js`, `package.json`, каталог **`node_modules`** (после `npm install` локально).
5. Включить **публичный HTTP-вызов** и скопировать **URL функции** в файл **`config.js`** на сайте (`FORM_SUBMIT_URL`).
6. Задать **переменные окружения** (см. ниже).

Если в VK Cloud у вас нет «функций» в привычном виде, используйте **контейнер** или **ВМ** с Node.js — тогда этот же код нужно обернуть в маленький HTTP-сервер; для типового сценария «форма → почта» удобнее именно serverless-функция, если она есть в тарифе.

## Переменные окружения

| Переменная | Пример | Описание |
|------------|--------|----------|
| `SITE_ORIGIN` | `https://org.github.io/repo` | Базовый URL сайта **без** слэша в конце (редирект после отправки). |
| `MAIL_TO` | `lovetattooandmoney@vk.com` | Куда приходят заявки (в проекте сейчас тестовый ящик; контакты на страницах сайта не трогали). |
| `SMTP_USER` | `bot@yandex.ru` | Логин SMTP. |
| `SMTP_PASSWORD` | `xxxx` | Пароль приложения почты (не обычный пароль). |
| `SMTP_FROM` | *(необязательно)* | От кого письмо; по умолчанию = `SMTP_USER`. |
| `SMTP_HOST` | `smtp.yandex.ru` | По умолчанию Яндекс; для Mail.ru — `smtp.mail.ru`. |
| `SMTP_PORT` | `465` | Обычно 465 (SSL). |
| `SMTP_SECURE` | `true` | `false` только если провайдер требует без SSL на порту (редко). |

## Сборка архива

В каталоге `function/`:

```bash
npm install
```

В zip кладите **содержимое** `function/`, чтобы в корне архива лежали `index.js`, `package.json`, `node_modules`.

## Проверка

Сайт открыт по тому же адресу, что в `SITE_ORIGIN`. Отправка формы → редирект с `?sent=1` → письмо на `MAIL_TO`.

## Сайт без PHP

Файл **`send.php`** в корне репозитория на **GitHub Pages не выполняется** — для продакшена используйте эту функцию в VK Cloud и **`config.js`**.
